
# Мост Bluetooth Debug для WearOS

[简体中文](README.md) | [English](README_en.md) | Русский

Приложение с foreground-сервисом на телефоне, которое возвращает отсутствующую на стороне телефона возможность Bluetooth ADB bridge для WearOS 3.

На часах по-прежнему используется системная опция "Отладка по Bluetooth". Это приложение отвечает за телефонную сторону:

- Выбор сопряженных часов
- Запуск RFCOMM Bluetooth-рукопожатия
- Публикация локального `adb-hub`
- Опциональное открытие TCP-порта для `adb connect` с этого устройства или из локальной сети

> [!WARNING]
> В версиях после WearOS 3 Bluetooth-отладка полностью удалена.
> Этот инструмент работает только с версиями WearOS от 2 до 3. 🤦
> Если в параметрах разработчика на часах нет "Отладка по Bluetooth", версия вашей системы не поддерживается.

## Текущие возможности

- Просмотр сопряженных Bluetooth-устройств на телефоне и выбор целевых часов
- Запрос разрешений "Устройства поблизости / Подключение Bluetooth / Уведомления" при холодном старте
- Постоянная работа в фоне через foreground-сервис
- Опциональный TCP-порт прослушивания, по умолчанию `44444`, можно задать `1-65535`
- Отдельные строки статуса для службы, хоста и цели с цветовой индикацией
- "Информация об endpoint" и "Команды подключения" по умолчанию свернуты и могут быть развернуты
- Поддержка копирования по нажатию и кнопкой для endpoint и команд подключения
- Ошибки рукопожатия, занятый порт, нехватка разрешений и другие проблемы показываются в UI и уведомлении

## Требования

- Android-телефон
- Часы Wear OS, уже сопряженные с телефоном по Bluetooth
- На часах в параметрах разработчика включена "Отладка по Bluetooth"

Для сборки приложения:
- JDK 11 или выше
- Рабочее окружение сборки Gradle

## Сборка

Windows:

```powershell
.\gradlew.bat assembleDebug
```

macOS / Linux:

```bash
./gradlew assembleDebug
```

Путь к debug APK:

```text
app/build/outputs/apk/debug/app-debug.apk
```

Готовые APK доступны на странице [Releases](https://github.com/SteveZMTstudios/WearOS-bluetooth-adb-wrapper/releases/).

## Использование

1. Установите и запустите приложение на телефоне.
2. При первом холодном запуске выдайте необходимые разрешения.
3. На главном экране выберите целевые часы.
4. Если нужен TCP, включите переключатель и задайте порт прослушивания.
5. Включите "Отладка по Bluetooth" в системных настройках часов.
6. Нажмите "Запустить мост".
7. Следите за строками состояния: "Служба / Хост / Цель".
8. При необходимости разверните "Команды подключения" и скопируйте содержимое.

## Два способа подключения

### 1. Классический `adb-hub`

Телефон создает локальный abstract socket:

```text
localabstract:/adb-hub
```

После подключения компьютера к телефону (USB или беспроводной режим) выполните:

```bash
adb forward tcp:44444 localabstract:/adb-hub
adb connect 127.0.0.1:44444
```

Если вы изменили TCP-порт, замените `44444` на свой.

### 2. TCP-прослушивание

После включения "Включить прослушивание TCP" приложение начинает слушать указанный порт телефона и показывает доступный адрес в "Информация об endpoint".

Это позволяет инструментам на телефоне (например, Termux) подключаться к часам по IP-адресу без обязательного Wi-Fi на телефоне, а также подключаться другим устройствам в той же локальной сети.

Пример:

```bash
adb connect 192.168.1.23:44444
```
